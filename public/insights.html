Python: Mutable Default Arguments Python

One of the most common gotchas in Python interviews. Default arguments are evaluated only once at function definition time, not each time the function is called.

# ‚ùå WRONG
def append_to(num, target=[]):
    target.append(num)
    return target

print(append_to(1)) # [1]
print(append_to(2)) # [1, 2] - Surprise! It kept the old list.

# ‚úÖ CORRECT
def append_to(num, target=None):
    if target is None:
        target = []
    target.append(num)
    return target
Why? Lists are mutable. If you use a mutable object as a default value, it persists across calls. ALWAYS use None as default for mutable types.

Java: String Pool & Equality Java

Understanding the difference between == and .equals() is crucial.

String s1 = "Hello";
String s2 = "Hello";
String s3 = new String("Hello");

System.out.println(s1 == s2);      // true (Same reference in Pool)
System.out.println(s1 == s3);      // false (Different objects)
System.out.println(s1.equals(s3)); // true (Same content)
Tip: Always use .equals() for string comparison. == compares memory references, which is rarely what you want for content.

C++: Vector Reallocation C++

Pushing to a vector can be expensive if it triggers a resize. Pointers to elements may become invalid. 

vector v = {1, 2, 3};
int* p = &v[0];

// Pushing many elements might cause reallocation
for(int i=0; i<100; i++) v.push_back(i);

// *p might now be garbage! The vector moved to new memory.
Optimization: Use v.reserve(n) if you know the approximate size beforehand to prevent multiple reallocations and improve performance.

JavaScript: Event Loop & Closure JS

The classic interview question involving `setTimeout` and loops.

// ‚ùå What does this print?
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100);
}
// Prints: 3, 3, 3 (Because 'var' is function-scoped)

// ‚úÖ Fix with 'let' (Block scoped)
for (let i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100);
}
// Prints: 0, 1, 2
Insight: `var` assumes function scope, so the loop finishes before the timeout runs, leaving `i` at 3. `let` creates a new binding for each iteration.
